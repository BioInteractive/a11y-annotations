Focus Order

  • All focusable components receive focus in an order that preserves meaning and operability.
  • Focus order will typically maintain a left-to-right, top-to-bottom sequence.
  • Focus must be sent to dialogs, stay trapped within them while open, and return to the triggering element when closed.
  • When moving the user's focus in response to activating a link or control, the next tab stop must move focus to the correct focusable element.

  1. Visual Inspection: Examine your Figma design annotations carefully, paying attention to the layout and order of interactive elements. Ensure that the order in which elements are presented in the design corresponds to a logical and meaningful sequence.
  2. Tab Order Annotation: Include annotations or notes within your Figma design that describe the expected tab order or focus order. You can use text or comment tools to provide explanations for the intended flow of keyboard focus.
  3. Interactive State Annotations: If your design includes interactive states (e.g., menus, modals, pop-ups), annotate these states to indicate how the focus should behave when these elements are triggered and opened. Describe how the focus should move within and return from these states.
  4. Keyboard Navigation Flow: Review the flowchart or documentation you've created to describe the expected keyboard navigation flow. Ensure that it aligns with your design annotations and that the focus order is logical.
  5. Documentation for Developers: If you're sharing the Figma design with developers, include additional documentation that outlines the desired focus order and behavior. This can serve as a reference for developers during implementation.

  • The tab order does not follow a logical sequence.
  • Review and adjust the tab order to ensure it follows a left-to-right, top-to-bottom flow that preserves meaning and operability.
  • Ensure that when interacting with links or controls, the next tab stop moves focus to the correct focusable element in a predictable manner.
  • If there are dialogs or pop-ups, confirm that focus is trapped within them when open and returns to the triggering element when closed.
