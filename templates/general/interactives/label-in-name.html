Label in Name

2.5.3 Label in Name The text shown on a visible label for a user interface component should match or be part of the component's accessible name (e.g., used by screen readers).

  • Consistent labels: The accessible name and visible label of a control should match. It is required that the accessible name contain the exact text string as the text visibly labeling this content.
  • Screen reader compatibility: Make sure the label spoken by screen readers matches what sighted users see on screen.
  • Reducing confusion: This alignment reduces confusion for users relying on voice commands or screen readers.
  • Easy interaction: Voice command users benefit when the visual label matches the name used by assistive technologies, making commands easier to understand and follow.

Design Considerations in Figma

  • Ensure Label Alignment: Design the interface so that the visible label text matches or is included in the accessible name for all elements like buttons, links, and inputs.
  • Consistent Naming: Clearly document how labels should be consistent across visible elements and the accessible name to ensure seamless interaction for users of assistive technologies.
  • Voice Command Friendly: Ensure that any design for voice-controlled interfaces matches what the user sees with what assistive technology announces.

By ensuring labels in your designs align with their accessible names, you help developers implement accessible content.

  1. Visual Inspection: Check whether visible labels pose potential issues for the accessible name, paying special attention to interactive elements like buttons and form inputs.
  2. Annotation for Labels: Annotate the designs specifying how the accessible names of components should include the text from their visible labels.
  3. Documentation: Include notes that describe which UI components need accessible names aligned with visible labels. Provide this information for the developer hand-off phase.

Label in Name

Ensure the following for any interactive elements with visible labels:

  • The visible label should match the accessible name and must at minimum be included in the accessible name.
  • Visible labels and accessible names should be consistent to avoid confusion.
  • Include the visible label in the aria-labelledby or aria-label attribute for components, ensuring accessible interaction.
  • Consider using the aria-describedby attribute for components, ensuring the accessible name will not be overridden with additional text must be associated.

Implementing Label in Name Accessibility

  • Match Visible and Accessible Names: Ensure that the visible label matches or is included in the element's accessible name. For example, if a button says “Submit,” the accessible name should also include “Submit.”
  • Use ARIA Effectively: Apply aria-labelledby or aria-label to associate the visible label with the accessible name of interactive elements.
  • Test with Screen Readers: Regularly test interactive elements with screen readers to ensure the correct name is announced when the element is focused.
  • Support Voice Commands: Ensure users who rely on voice commands can trigger actions by saying what they see. Aligning labels with names will facilitate smoother interactions for these users.
  • Consistent User Experience: Ensure that accessible names, especially for buttons and form controls, are consistent across the app to provide a seamless experience for all users.

Following these guidelines ensures that your implementation aligns with WCAG 2.5.3, providing an accessible experience for users who depend on screen readers and voice commands.
