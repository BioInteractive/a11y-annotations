Touch Target

  • 2.5.8 - Target Size (Minimum) The size of the target for pointer inputs must be at least 24 by 24 CSS pixels.
  • Except where:
    • Spacing: Small touch targets must not overlap with each other or larger targets to avoid accidental taps.
    • Equivalent Controls: If there's another way to do the same action with larger touch targets on the same page, it should be provided.
    • Inline Targets: Even if a touch target is within a sentence or constrained by nearby text, it should still be comfortably sized for tapping.
    • User Agent Control: The device (e.g., phone or tablet) should determine touch target size, not the website or app.
    • Essential Presentation: If a specific touch target presentation is necessary for clarity or legal reasons, it should meet the minimum size requirement.

  1. Visual Inspection: Examine your Figma design annotations carefully, look at the interactive elements (like links, buttons and icon buttons).
  2. Is their visual footprint meeting the minimum size requirements? Select the element on the Figma Board and check the design panel width and height measurements.
  3. For the interactive elements that are too small visually, draw a touch target area and include notes for the designer to adjust the sizes before dev hand-off.
  4. Number and add additional notes for developers with touch target requirements for any elements that were originally flagged for potential issues.

  • Small interactive element below 24x24 CSS pixels and insufficiently spaced from other controls.
  • Text links below 24x24 CSS pixels and and insufficiently spaced from other controls.
  • Small form input fields below 24x24 CSS pixels and insufficiently spaced from other controls.
