Keyboard

  • All interactive elements must receive focus and function with the keyboard.
  • All functionality of the content can be operated via a keyboard interface without requiring specific timings for individual keystrokes.

  • If keyboard focus can be moved to a component of the page using a keyboard interface, the focus can be moved away from that component using only a keyboard interface, and the component does not trap keyboard focus
  • Focus must not be trapped with forward Tab navigation or backwards Tab navigation. (Shift + Tab)

  1. Visual Inspection: Examine your Figma design annotations carefully, paying attention to the presence and behavior of interactive elements. Ensure that interactive elements, including buttons, links, and form controls, are visible and can be identified in the design.
  2. Keyboard Operability Annotation: Include annotations or notes within your Figma design that describe the expected keyboard operability of interactive elements. Indicate that all interactive elements must be fully functional and operable using only the keyboard, without requiring specific timings for keystrokes.
  3. Documentation for Developers: ITo prepare for Figma design hand-off to developers, include additional notes and/or documentation that outline the keyboard accessibility requirements. This documentation can serve as a reference for developers to ensure that all keyboard-related requirements are met during implementation.

  • Lack of Keyboard Focus: Interactive elements like buttons, links, and form controls don't receive keyboard focus when navigating the page, making them inaccessible via keyboard.
  • Interactive Element Function: Element functions for pointer inputs but not for keyboard events, additional event handlers may not have been implemented.
  • Keyboard Traps: Certain design components, such as modal dialogs or dropdown menus, must trap keyboard focus, and there is not a keyboard method for easily navigating away or untrapping focus.
  • Non-Functional Keyboard Shortcuts: Keyboard shortcuts that conflict with standard browser or assistive technology shortcuts can disrupt keyboard navigation and functionality.
