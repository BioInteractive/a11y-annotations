Bypass Blocks

2.4.1 Bypass Blocks: A mechanism is available to bypass blocks of content that are repeated on multiple web pages. This requirement is essential for improving navigation for users who depend on screen readers or keyboard navigation by allowing them to directly access the primary content without navigating through repetitive sections.

The Bypass Blocks guideline addresses the need for efficient navigation options to help users avoid repetitive elements. These can include main navigation menus, footer links, carousels, or sidebars that may otherwise be cumbersome to navigate through on every page. By enabling a skip link or similar mechanism, users can bypass these repetitive blocks and immediately focus on the content they need.

Common implementations include a "skip to main content" link positioned as the first focusable element on the page, often visually hidden until it receives keyboard focus. Additionally, HTML5 landmarks (like <header>, <nav>, <main>, and <footer>) and role attributes (such as role="main" or role="navigation") help assistive technologies identify key sections of a page, further supporting navigation and enhancing understanding of page structure. For more information visit Landmarks.

For users who rely on keyboard navigation or screen readers, the ability to bypass repetitive blocks of content is crucial for reducing the time and effort required to locate main content. Without these mechanisms, users may need to tab through multiple elements just to reach the core information, leading to a frustrating experience. Implementing bypass blocks is a key step in creating a more accessible web experience that meets the needs of all users, especially those with disabilities.

Designing for Keyboard Accessibility

  • Designing Skip Links: Include visual representations of skip links in your Figma designs displaying them with brand colors in focus state. This not only aids in early planning for keyboard navigation but also highlights repetitive blocks of content that require skip links during the design phase.
  • Identifying Common Repetitive Blocks of Content: Recognize common areas where skip links are required and/or beneficial for usability. This includes:
    • Repetitive navigational elements like top navigation bars, sidebar menus or secondary navigational menus.
    • Recurring elements like carousels when they contain multiple items or links, long lists of links or lengthy introductory content especially. (A skip link is only required if the content is repetitive across multiple pages but highly recommended if there are multiple tab stops that will make navigational difficult for keyboard and/or screen reader users.)

  1. Visual Identification of Skip Link: Confirm the presence of a styled skip link at the top of each page or before repetitive components (e.g., primary navigation, carousels, or repeated in-page navigation elements). If missing, request an example from the designer that includes a distinct, visible focus and activation state.
  2. Annotation for Skip Link Functionality: Specify the skip link’s purpose and function in annotations, highlighting its role in allowing keyboard users to bypass repetitive content and navigate directly to the main content. Ensure it is the first interactive element in the keyboard tab order and that pressing "Enter" shifts focus to the main content.
  3. Focus State Requirements: Confirm that the skip link’s focus state is visibly distinct in design to aid keyboard-only users. This state should meet WCAG 2.1 AA contrast requirements (minimum 3:1 ratio) using styles like outline or color change.
  4. Documentation for Developers: Provide notes specifying skip link implementation requirements:
    • The skip link should be the first tab-able element for keyboard navigation.
    • Pressing "Enter" on the skip link should immediately shift focus to the main content area.

Skip Link

  • Must be a link <a> with a valid href to the <main> element's ID.
  • Targeted main element after navigation should be either in a <main> or sectioning element with role="main".
  • Link should be visually hidden with CSS styling until receiving focus and then visually hidden again when focus leaves.
  • Main element should have a tabindex="-1" so that when the link is activated the element will be focusable but not in regular focus order.

Implementing Skip Links

To implement an accessible skip link, start by creating a link element (<a>) with an href attribute that points to the main content section's unique ID (e.g., <main id="main-content">). This setup enables users who navigate by keyboard to skip repetitive elements, like navigation, and jump directly to the main content area by pressing "Enter" when the skip link is in focus.

The target destination for the skip link should be either a <main> element or another appropriate sectioning element, such as <section> or <div>, with the attribute role="main". This semantic markup identifies the destination as the page’s primary content for screen readers and assistive technology, helping users understand its importance.

To keep the skip link visually hidden until it's needed, use CSS to move it off-screen by default. Apply styles such as position: absolute; top: -40px; left: 0; along with a high-contrast background color, such as #005a9c, to make it accessible when in focus. When the skip link receives focus, CSS in the :focus state should change the top property to 0 to bring it into view, allowing keyboard-only users to see and activate it. Here’s a sample CSS snippet:


                          .skip-link {
                            position: absolute;
                            top: -40px;
                            left: 0;
                            background-color: #005a9c; /* or another high-contrast color */
                            color: white;
                            padding: 8px;
                            z-index: 100;
                            text-decoration: none;
                          }
                          .skip-link:focus {
                            top: 0;
                            position: absolute;
                          }

Finally, add tabindex="-1" to the main content container (e.g., <main id="main-content" tabindex="-1">). This attribute ensures that the main content can receive focus when the skip link is activated but doesn’t add it to the default tab order, preserving a smooth keyboard navigation experience. By following these steps, you ensure the skip link meets WCAG 2.1 AA accessibility standards and enhances usability for keyboard and screen reader users.
