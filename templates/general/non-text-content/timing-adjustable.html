Timing Adjustable

2.2.1 Timing Adjustable For time limits set by the content, users must have options to manage these limits effectively, ensuring accessibility and user control.

  • Turn Off: Users can disable the time limit before encountering it.
  • Adjust: Users can adjust time limits before they start, up to at least ten times the default duration.
  • Extend: Users receive a warning before a time limit expires and can easily extend it at least ten times by performing a simple action.
  • Real-time Exception: Time limits integral to real-time events (e.g., auctions) where no alternative exists.
  • Essential Exception: Time limits that are crucial and where extension would invalidate the activity.
  • 20 Hour Exception: Time limits exceeding 20 hours.

Designing for Timing Adjustable Accessibility

  • Visual Timers and Warnings: Design visual cues such as timers or warning messages to alert users before a time limit expires, ensuring they are aware of the need to extend the time if possible.
  • Adjustment Controls: Incorporate design elements that allow users to adjust or turn off time limits. Consider sliders, toggles, or settings menus for user-controlled time adjustments.
  • Extension Options: Clearly indicate options for extending time limits within the interface, making it easy for users to find and use these options to prevent stress or confusion.
  • Documentation and Guidelines: Provide detailed documentation and guidelines within your design annotations for developers, ensuring they understand how to implement the adjustable timing features correctly.

  2. Check for timer controls: Ensure there are designs for user controls to adjust, extend, or turn off time limits, including visual cues for these options.
  3. Look for warnings before time expiration: Confirm the presence of designs for warning messages alerting users before a time limit expires.
  4. Inspect for multiple extension opportunities: Verify designs include options for users to easily extend time limits multiple times without complex interactions.
  5. Assess for essential time limits: Differentiate between components with essential time limits and those where adjustments could be made, marking potential areas for design modifications.

Timing Adjustable

  • If a time limit is set and it is shorter than 20 hours, it must be adjustable/extendable.
  • Warning: Ensure timing is adjustable or extenable through an accessible mechanism.
  • Recommendation: Use an accessible warning dialog with options to extend time.

Developing with Timing Adjustable Considerations

  • Implement Adjustable Timers: Ensure all time-limited functionalities allow users to adjust, extend, or disable timing. Provide options in the UI for users to select their preferred time settings.
  • Warning Messages: Code warning messages to alert users before a time limit expires, with an easy mechanism to extend the time directly from the alert.
  • Accessible Controls: Develop accessible controls for adjusting time limits, ensuring they can be operated with keyboard alone and are clearly labeled for assistive technology.
  • Document Essential Limits: For time limits that are essential and cannot be adjusted, provide thorough documentation in the code to explain the necessity of the time constraint.
  • Long Duration Timeouts: For activities with a timeout longer than 20 hours, ensure mechanisms are in place to preserve user data and state.

Following these guidelines helps developers ensure web content is accessible and compliant with WCAG 2.2.1 Timing Adjustable criteria, catering to users' needs for flexible interaction times.
