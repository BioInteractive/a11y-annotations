Tables

1.3.1 Info and Relationships: Tables must present information in a way that preserves relationships between data cells and their headers. This ensures that the structure and relationships are programmatically determined or available in text.

Proper use of table elements like <th> for headers and <td> for data cells. For more complex tables, explicit associations may be needed using scope, id, and headers attributes.

Tables must have accessible names with <caption> tags, an aria-label or an aria-labelledby equal to the ID of the labelling text.

Example of a complex table with multiple headers
Item Details
Description Price
Item 1 Item 1 detailed description $15
Item 2 Item 2 detailed description $25

Note: These are simplified examples. Real-world tables might require additional accessibility features based on their complexity and content.

  1. Start by identifying all the table elements including headers and data cells. Understand the relationship between these elements.
  2. Decide if the table headers clearly labels the associated data cells.
  3. If the table is complex add a sticky note annotation to give additional information about table scopes, ids and headers.
  4. Note that tables must have a caption or aria-labelledby pointing to labelling text to ensure the table has an accessible name.
