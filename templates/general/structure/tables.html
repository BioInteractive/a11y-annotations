Tables

1.3.1 Info and Relationships: Tables must present information in a way that preserves relationships between data cells and their headers. This ensures that the structure and relationships are programmatically determined or available in text.

Proper use of table elements like <th> for headers and <td> for data cells. For more complex tables, explicit associations may be needed using scope, id, and headers attributes.

Tables must have accessible names with <caption> tags, an aria-label or an aria-labelledby equal to the ID of the labelling text.

When designing tables in Figma, ensuring accessibility is crucial for all users, including those using assistive technologies. Here are key considerations for designers:

  1. Clear Table Headers: Designate clear headers for both columns and rows. Headers should succinctly describe the data below them or to their right. This helps users understand the structure and content of the table at a glance.
  2. Visual Distinction for Headers: Style table headers differently than data cells to visually distinguish them. Consider using bold text, larger font sizes, or background colors to achieve this.
  3. Consistent Gridlines and Spacing: Use consistent gridlines and spacing within the table to enhance readability. Clear separation between rows and columns aids users in tracking information across the table.
  4. Logical Structure: Design tables with a logical structure in mind. Information should flow in a predictable manner, typically from left to right and top to bottom, mirroring the natural reading order.
  5. Use of Zebra Stripes: (Usability / Not Required) Consider using zebra stripes (alternating row colors) to improve the readability of large tables. This can help users keep their place as they move through the table data.
  6. Include Sort and Filter Options: (Usability / Not Required) If the table will be interactive, design clear indicators for sorting and filtering options. This allows users to navigate and find the information they need more efficiently.
  7. Accessible Color Schemes: Ensure that the color scheme used for tables has sufficient contrast between text and background colors, as well as between different table elements, to meet WCAG color contrast ratios.

By following these guidelines, you'll contribute to creating designs that support accessibility and ensure a better experience for all users, including those relying on assistive technologies to navigate and understand table content.

Ensuring the accessibility of tables is critical for developers, as it allows all users, including those using assistive technologies, to understand and navigate the content efficiently. Consider these guidelines when coding tables:

  1. Clear Table Headers: Use semantic markup to define <th> elements for table headers, ensuring they are descriptive of the content they relate to.
  2. Accessible Names: Provide tables with accessible names using the <caption> tag or aria-labelledby, offering a clear description of the table's content and purpose.
  3. CSS for Visual Distinction: Apply CSS styles to differentiate table headers from data cells visually. This can include font weight, background color, or text alignment.
  4. Consistent Grid Lines and Spacing: Use CSS to maintain consistent grid lines and spacing within the table to enhance readability.
  5. Logical Structure: Ensure the table's structure is logical and straightforward, with related data grouped together and a logical flow from left to right and top to bottom.
  6. Sorting and Filtering: Include options for sorting and filtering table data to assist users in finding the information they need more efficiently.
  7. Complex Tables: For more complex tables, utilize scope attributes to specify whether a header applies to a row, column, or group of rows or columns. Use ids and headers attributes to associate data cells with header cells explicitly.

By following these guidelines, developers can create tables that are not only visually appealing but also accessible, ensuring that all users can access and understand the table content.
