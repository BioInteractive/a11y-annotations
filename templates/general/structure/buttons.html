Buttons

1.3.1 Info and Relationships: Information, structure, and relationships conveyed through presentation can be programmatically determined or are available in text.

A button is an interactive user interface element, typically created using the HTML <button> element, designed to trigger specific actions or functions when interacted with by users, triggering actions, and engagement within digital interfaces.

Buttons must be preceivable as buttons, must have an accessible name, must be in focus order and must function equally for all users.

Examples of Common Buttons

  • - Used in forms to submit data.
  • - Cancels actions or forms.
  • - Initiates the login process.
  • - Logs users out.
  • - Adds products to a cart.
  • - Downloads files or resources.
  • - Executes search queries.
  • - Saves changes made.
  • - Enters an edit mode.
  • - Deletes selected items or content.

  1. Use this A11y Annotation Kit.
  2. Identify all UI elements that perform actions.
  3. During phase one, ask questions if neccesary to understand functionality.
  4. During phase two in preparation for developer hand-off, add a sticky note annotation to define each button element as a button.
  5. Most buttons will require further annotating for the developer hand-off phase. Please see interactives and the How to Annotate for more detailed instructions on how to number the elements and write annotations specific for more complex components.
  6. If the button does not have a visible text label it is recommended to use the button annotation tag with the additional label to ensure the button name will be communicated to the developer for programming.
