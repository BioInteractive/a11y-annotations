Lists

1.3.1 Info and Relationships: Information, structure, and relationships conveyed through presentation can be programmatically determined or are available in text.

Semantically marking up lists helps assistive technologies (such as screen readers) understand the structure and purpose of the content. For users who rely on these technologies, a list marked up as a list provides context and conveys that the items are related.

Visually presented lists must be in semantic list markup to ensure the information and relationships are programmatically determinable.

  • Unordered List: A list of items that are listed in no particular order. Use <ul> to create an unordered list. List items are typically preceded by bullet points, though the visual representation may vary.
  • Ordered List: A list of items that are listed in a specific order, typically indicated by numbers, letters, or other markers. Use <ol> to create an ordered list. List items are automatically numbered or lettered.
  • Definition List: A list of terms and their corresponding definitions. Each term is represented using <dt> (definition term), and its definition is represented using <dd> (definition description). Use <dl> to create a definition list.
  • Nested Lists: Lists can be nested within other lists, allowing you to create hierarchical structures. Use <ul> or <ol> within another list item to create nested lists.

Examples

Unordered List Type

  • Apples
  • Bananas
  • Oranges

Ordered List Type

  1. Start with the basic stance.
  2. Step forward with your right foot.
  3. Twist to the left and raise your left hand.
  4. Complete the routine with a spin.

Definition List Type

Programming Languages:
Java - An object-oriented programming language.
Python - Known for its simplicity and readability.
Planets in the Solar System:
Earth - The third planet from the Sun.
Mars - Known as the "Red Planet."

Nested Lists Type

  • Travel Destinations:
    • Europe
      • France
      • Italy
      • Spain
    • Asia
      • Japan
      • Thailand
      • India

  1. Use this A11y Annotation Kit.
  2. Identify all text the is grouped and styled as lists.
  3. Decide which list type would be most appropriate by the text content.
  4. Add a sticky note annotation by copying and pasting to define text as a list and indicate the correct list type.
  5. For nested lists use additional tags to tag each nested list with the correct list type individually.
