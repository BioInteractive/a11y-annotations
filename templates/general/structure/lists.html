Lists

1.3.1 Info and Relationships Information, structure, and relationships conveyed through presentation can be programmatically determined or are available in text.

Semantically marking up lists helps assistive technologies (such as screen readers) understand the structure and purpose of the content. For users who rely on these technologies, a list marked up as a list provides context and conveys that the items are related.

Visually presented lists must be in semantic list markup to ensure the information and relationships are programmatically determinable.

Note on VoiceOver: Voiceover has a known bug for not announcing unordered lists properly when either the list-style-type is set to none or display is set to inline. One solution is to use ARIA roles list and listitem to restore the semantics or another is to use CSS. Please see this article for eference and examples.

List Types

  • Unordered List: A list of items that are listed in no particular order. Use <ul> to create an unordered list. List items are typically preceded by bullet points, though the visual representation may vary.
  • Ordered List: A list of items that are listed in a specific order, typically indicated by numbers, letters, or other markers. Use <ol> to create an ordered list. List items are automatically numbered or lettered.
  • Definition List: A list of terms and their corresponding definitions. Each term is represented using <dt> (definition term), and its definition is represented using <dd> (definition description). Use <dl> to create a definition list.
  • Nested Lists: Lists can be nested within other lists, allowing you to create hierarchical structures. Use <ul> or <ol> within another list item to create nested lists.

Correctly coding lists is fundamental to web accessibility, ensuring compliance with WCAG 1.3.1. Lists that visually appear as such should be semantically marked up to reflect their structure and function. Follow these guidelines for implementing lists:

  • Semantic Markup for Lists: Use the appropriate HTML list element for its purpose. An unordered list (<ul>) for items without a specific order, an ordered list (<ol>) for items that follow a sequence, and a description list (<dl>) for pairs of terms and descriptions.
  • Consistent List Styling: Ensure that lists are styled in a manner that reflects their hierarchy and function. Use CSS to visually distinguish between different types of lists and their nested levels.
  • Nesting Lists Properly: When lists are nested within other lists, make sure the nesting is logical and semantically correct. Each nested list should be contained within a list item (<li>) of the parent list.
  • Use of Description Lists: Employ description lists (<dl>) for grouping together terms and their descriptions. This is particularly useful for glossaries, metadata presentations, or any scenario where a term needs to be associated with a description.
  • Accessible List Markup: Beyond using the correct list type, ensure that additional attributes like ARIA roles and properties are applied as necessary to enhance the accessibility of complex list structures or when lists are used for navigation purposes.

By adhering to these guidelines, developers can create list structures that are both meaningful and accessible, promoting better navigation and understanding of the content for all users, including those using assistive technologies.
