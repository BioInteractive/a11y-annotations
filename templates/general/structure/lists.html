Lists

1.3.1 Info and Relationships requires that information, structure, and relationships conveyed visually—such as through layout, headings, or grouping—can also be understood programmatically or are available in text.

Briefing and Intent

Lists communicate that a group of items is related. When lists are only styled visually but not marked up semantically, screen reader users may miss the grouping and context. This criterion ensures that lists are properly coded so assistive technologies can understand the structure and convey it to users accurately.

Benefits

Semantic list markup helps screen reader users understand that items are part of a set, making content easier to comprehend and navigate. Properly structured lists also improve accessibility for users who rely on voice navigation and enhance content clarity for all users.

Key Requirements

  • Use <ul> for unordered (bulleted) lists and <ol> for ordered (numbered) lists.
  • Use <dl>, <dt>, and <dd> to structure definition lists.
  • Ensure list items are wrapped in <li> elements, even if custom bullets or spacing is applied visually.
  • Use nested list elements properly to represent sublists or hierarchical structures.
  • Avoid relying on visual styling alone (e.g., dashes, line breaks) to imply list relationships.

Designing Accessible Lists

  • Use Bullet Points Thoughtfully: If content visually looks like a list, design it as a list. Use bullet points for grouped ideas and numbers for steps or sequences.
  • Avoid Decorative Icons for Bullets: Stick to simple, consistent bullet styles. Decorative or branded icons can cause confusion and reduce scannability.
  • Design with Semantic Intent: Although you are designing visually, remember that developers will mark up lists semantically. Proper list markup helps screen reader users understand that items are grouped and related.

  1. Use this A11y Annotation Kit.
  2. Identify all text the is grouped and styled as lists.
  3. Decide which list type would be most appropriate by the text content.
  4. Add a sticky note annotation by copying and pasting to define text as a list and indicate the correct list type.
  5. For nested lists use additional tags to tag each nested list with the correct list type individually.

Correctly coding lists is fundamental to web accessibility, ensuring compliance with WCAG 1.3.1. Lists that visually appear as such should be semantically marked up to reflect their structure and function. Follow these guidelines for implementing lists:

  • Semantic Markup for Lists: Use the appropriate HTML list element for its purpose. An unordered list (<ul>) for items without a specific order, an ordered list (<ol>) for items that follow a sequence, and a description list (<dl>) for pairs of terms and descriptions.
  • Consistent List Styling: Ensure that lists are styled in a manner that reflects their hierarchy and function. Use CSS to visually distinguish between different types of lists and their nested levels.
  • Nesting Lists Properly: When lists are nested within other lists, make sure the nesting is logical and semantically correct. Each nested list should be contained within a list item (<li>) of the parent list.
  • Use of Description Lists: Employ description lists (<dl>) for grouping together terms and their descriptions. This is particularly useful for glossaries, metadata presentations, or any scenario where a term needs to be associated with a description.
  • Accessible List Markup: Beyond using the correct list type, ensure that additional attributes like ARIA roles and properties are applied as necessary to enhance the accessibility of complex list structures or when lists are used for navigation purposes.

By adhering to these guidelines, developers can create list structures that are both meaningful and accessible, promoting better navigation and understanding of the content for all users, including those using assistive technologies.
