Language

3.1.1 Language of Page mandates that the primary language of each webpage be programmatically determined. This is crucial for users who rely on text-to-speech software or screen readers, ensuring that content is read and pronounced correctly.

3.1.2 Language of Parts requires that the language of each passage or phrase in the content be programmatically determinedâ€”except for proper names, technical terms, or words that have become part of the vernacular of the surrounding text. This is especially important on pages with multiple languages.

Briefing and Intent

Together, these criteria ensure that both the overall page and any localized sections are accurately identified for assistive technologies, enabling correct pronunciation and intonation for a global audience.

Key Requirements

  • For Language of Page: Set the lang attribute on the <html> tag to declare the primary language.
  • For Language of Parts: Use the lang attribute on elements containing text in a language different from the primary language.
  • Ensure any change in language is clearly marked so assistive technologies can adjust pronunciation accordingly.

Examples

  • Single Language Page: <html lang="en"> for an English-only page.
  • Multilingual Content: <html lang="en"> with specific sections marked, e.g., <p lang="fr"> for French passages.

Designing for Language Specification

  • Clear Documentation: Label Figma boards and design assets with the intended primary language and any alternate language sections.
  • Visual Cues for Language Changes: When designing multilingual interfaces, annotate sections where the language changes so developers know where to apply additional lang attributes.
  • Consistency in Design: Ensure that design elements, such as headings and body text, reflect the declared language through consistent styling and clear labeling.

  1. Review Design Artifacts: Examine Figma prototypes to verify that the intended language settings and changes are clearly annotated.
  2. Annotate Discrepancies: Document any areas where language attributes may be missing or inconsistent, and provide clear notes for developers.
  3. Confirm Language Flow: Ensure that the visual presentation aligns with the underlying language markup for seamless assistive technology interpretation.

Implementing Language Specifications

  • Default Language: Set the primary language for the page by adding the appropriate lang attribute to the <html> tag.
  • Marking Language of Parts: For content that switches languages, add lang attributes to the respective elements.
  • Consistent Metadata: Ensure that metadata such as keywords and descriptions align with the declared language.
  • Regular Validation: Review and update language settings periodically as content evolves.
