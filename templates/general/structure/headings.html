Headings

1.3.1 Info and Relationships: Information, structure, and relationships conveyed through presentation can be programmatically determined or are available in text.

Properly structured headings are essential for web accessibility. They provide a hierarchical structure to your content, making it easier for all users, including those who rely on screen readers, to navigate and understand your web page.

Heading levels should be used sequentially, starting with <h1> for the main page title and following with <h2>, <h3>, and so on, to represent subheadings. Avoid skipping levels in the hierarchy, as this can lead to confusion for screen reader users.

Nesting headings correctly is crucial for maintaining a logical and meaningful structure on your web page. Incorrect nesting can result in WCAG 1.3.1 failures. Follow these guidelines:

  1. Use <h1> for the Main Heading: The main heading of your page should always be <h1>. It represents the primary topic or title of the page.
  2. Nest Subheadings Sequentially: When adding subheadings, use <h2> for the first-level subheading, <h3> for second-level, and so on. Each subheading should be nested under the previous heading to create a logical hierarchy.
  3. Avoid Skipping Heading Levels: Do not skip heading levels. For example, do not jump from <h2> to <h4> without using <h3> in between. Skipping levels can cause confusion and accessibility issues.
  4. Use Semantic HTML: Whenever possible use semantic HTML elements but if not possible you may use role="heading" and the appropriate number level indicated with aria-level="_".
