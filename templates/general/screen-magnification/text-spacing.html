Text Spacing

1.4.12 Text Spacing no loss of content or functionality occurs by setting all of the following and by changing no other style property:

  • Line height (line spacing) to at least 1.5 times the font size;
  • Spacing following paragraphs to at least 2 times the font size;
  • Letter spacing (tracking) to at least 0.12 times the font size;
  • Word spacing to at least 0.16 times the font size.

Text Spacing Design Considerations

  • Adjust Text Properties: When designing in Figma, set line height to at least 1.5 times the font size, paragraph spacing to at least 2 times the font size, letter spacing (tracking) to at least 0.12 times the font size, and word spacing to at least 0.16 times the font size to meet WCAG 2.1 Success Criterion 1.4.12 Text Spacing.
  • Test Text Adaptability: Simulate user-defined text spacing adjustments in your designs to ensure content remains legible and functional. This might involve manually adjusting text properties or using plugins that simulate text spacing adjustments.
  • Document Design Choices: Include notes or a separate document in your Figma file outlining the text spacing guidelines followed. This will help developers understand the rationale behind text property choices, ensuring a smoother transition from design to development.

Implementing these text spacing considerations in your Figma designs will help ensure that content is accessible and legible for users who adjust text spacing for readability, aligning with WCAG 2.1.1 requirements and enhancing overall user experience.

Developer Guidelines for Implementing Text Spacing Accessibility

  • Apply CSS for Text Spacing: Use CSS properties to manage text spacing according to WCAG guidelines. This includes setting line height to at least 1.5 times the font size, paragraph spacing to at least 2 times the font size, letter spacing to at least 0.12 times the font size, and word spacing to at least 0.16 times the font size.
  • Maintain Readability and Functionality: Ensure that adjustments in text spacing do not compromise content readability or functionality. Test your layouts with increased text spacing to prevent content clipping or overlap.
  • Testing and Validation: Test your website with various text spacing settings to ensure no loss of content or functionality. Use browser developer tools or CSS preprocessors to simulate different user settings and validate against WCAG criteria. The ARC Toolkit browser extension has a great Text Spacing measuring tool that can be used right in the browser while developing.

Adhering to these text spacing guidelines ensures accessibility for users needing adjustments for readability, aligning with WCAG 2.1.4 Text Spacing criteria.
