Reflow

1.4.10 Reflow Content can be presented without loss of information or functionality, and without requiring scrolling in two dimensions for:

  • Vertical scrolling content at a width equivalent to 320 CSS pixels;
  • Horizontal scrolling content at a height equivalent to 256 CSS pixels;
  • Except for parts of the content which require two-dimensional layout for usage or meaning

Designing for Reflow

  • Design Flexible Layouts: Create designs that adapt smoothly to different screen sizes, ensuring content reflows naturally without requiring users to scroll in two directions. Prioritize vertical layouts that can stack and wrap as needed.
  • Test at Small Viewports: In Figma, test your designs at a width of 320 CSS pixels to simulate a desktop view at 400% zoom. This is a critical check for users with low vision and helps ensure no content is clipped, cut off, or requires horizontal scrolling.
  • Use Consistent Breakpoints Visually: Plan how your layout should shift at common screen sizesâ€”mobile, tablet, and desktop. Consider how grids, columns, and components will adapt at smaller widths to maintain clarity and readability.
  • Design for All Orientations: Ensure your content looks good and remains functional in both portrait and landscape layouts. Prioritize flexibility in spacing, stacking, and content flow.
  • Support Scalable Content and Images: Use image containers and flexible spacing in your layouts to ensure that text and images resize or wrap properly at smaller screen widths or high zoom settings.

Designing with reflow in mind is one of the most effective ways to support users with low vision. By keeping your Figma layouts flexible and readable at all screen sizes, you help ensure accessibility across devices and zoom levels in alignment with WCAG 1.4.10 Reflow.

  1. Look for mobile size design boards and if they are not provided then ask the designer to provide them in 320px width.
  2. Check that the page does not require scrolling in two dimensions at a screen width of 320 CSS pixels and a screen height of 256 CSS pixels.
  3. Ensure that no meaningful content is lost or functionality is compromised when the page is viewed at these dimensions. Content in reflow must not be missing, cutoff, or overlapping.
  4. It is acceptable if meaningful content is presented in a different format as long as no meaning is lost, for example a footer list of links may be transformed into an accordion as long as all the same links are available to users on all screen sizes.
  5. Verify that content reflows and is usable without requiring two-dimensional scrolling, except for parts of content that require a two-dimensional layout, such as tables.

Reflow

  • Content or functionality is lost when the page is viewed at 320 CSS pixels width or 256 CSS pixels height.
  • Warning: Ensure that content reflows to adapt to different screen sizes and orientations.
  • Recommendation: Use responsive design techniques to ensure content reflows effectively at different screen sizes and orientations.

Implementing Reflow in Development

  • Responsive Design Techniques: Utilize CSS media queries to create responsive layouts that adapt to various screen sizes, ensuring content is accessible without two-dimensional scrolling.
  • Minimize Fixed Dimensions: Avoid using fixed width and height in CSS for elements. Instead, use percentages or viewport units to allow elements to adjust dynamically to screen sizes.
  • Testing on Various Devices: Test your implementation on devices with small screens (e.g., 320 CSS pixels in width and 256 CSS pixels in height) to ensure content reflows properly without requiring scrolling in two dimensions.
  • Adaptive Images and Containers: Implement techniques for images and containers that scale or hide appropriately to maintain content and functionality without the need for horizontal scrolling.

Following these guidelines ensures that your web content complies with WCAG 2.1.4 Reflow criteria, providing a better user experience across all devices and screen sizes.
