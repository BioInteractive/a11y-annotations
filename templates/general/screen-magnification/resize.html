Resize Text

1.4.4 Resize Text Except for captions and images of text, text can be resized without assistive technology up to 200 percent without loss of content or functionality.

  • Text Resize: Users must be able to resize text up to 200 percent.
  • Content Availability: All meaningful content must remain available and not be obscured or cut off due to resizing.
  • Functionality: The functionality of the website should not be compromised by the resizing of text.
  • Responsive Design: Web design should be responsive to accommodate text scaling while preserving content and functionality.

Designing for Text Resize Accessibility

  • Flexible Layouts: Use flexible layouts that adapt to text size changes. Ensure your design accommodates text scaling up to 200% without losing content visibility or functionality.
  • Avoid Fixed Sizes: Design with relative units (e.g., ems, rems) instead of fixed units (e.g., pixels) for text sizes, margins, and padding to support text resizing.
  • Content Flow: Ensure that text resizing does not cause content overlap or hide important information. Design elements should reflow to maintain content accessibility and readability.
  • Test with Different Text Sizes: Simulate text resizing in your design process to identify and address potential issues with content flow and accessibility.

By integrating these considerations into your design process, you help ensure that your website or application remains accessible and functional for users who need to resize text, in line with WCAG 2.1.4 Resize Text criteria.

  1. Inspect Text Elements: Review all text elements on the Figma board to ensure they are not rasterized images of text but editable text elements. Images of text won't resize in the same way as text elements.
  2. Responsive Design Check: Verify that the design layouts are responsive. This means that they should be able to adjust to different screen sizes and text sizes without losing content visibility or functionality. Ask for additional boards to demonstrate different screen sizes or text sizes, if applicable.
  3. If elements transform on smaller screen sizes verify that there is no loss of meaningful content and pay attention to new content that may require additional annotations. Examples: lists of footer links that transform into an accordion, info cards that tranform into a carousel, etc.

Developing for Text Resize Accessibility

  • Responsive Design: Implement responsive design techniques with CSS, ensuring layouts and elements adapt seamlessly when text is resized up to 200%.
  • Relative Units: Use relative units (em, rem, %) for text sizes, spacing, and layout dimensions to support dynamic resizing without breaking the page layout.
  • Media Queries: Employ CSS media queries to adjust layout and text properties for different levels of text enlargement, enhancing readability and ensuring content availability.
  • Testing and Validation: Regularly test your web pages set at 1280px width with text scaling up to 200% to ensure content remains accessible, functional, and without loss or overlapping of important information.

Applying these development practices ensures that your application or website remains usable and accessible for users needing to resize text, aligning with WCAG 2.1.4 Resize Text criteria.
