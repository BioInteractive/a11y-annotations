Form Group

  • 1.3.1 Info and Relationships Information, structure, and relationships conveyed through presentation can be programmatically determined or are available in text, ensuring that form fields and their labels are properly associated.
  • 4.1.2 Name, Role, Value For all user interface components, the name and role can be programmatically determined; states, properties, and values that can be set by the user can be programmatically set; and notification of changes to these items is available to user agents, including assistive technologies.
  • 3.3.2 Labels or Instructions Labels or instructions are provided when content requires user input.

Similar form fields must be grouped together to improve form comprehension for all users. This grouping makes it easier for users to identify related controls and allows them to concentrate on smaller, more manageable sections of the form, rather than attempting to understand the entire form at once.

Controls that must be grouped with a <fieldset> and a <legend> include:

  • Radio buttons
  • Checkboxes
  • Input fields for related information (e.g., first name, last name)
  • Select dropdowns
  • Date pickers

Grouping these controls with a <fieldset> and a descriptive <legend> provides clarity and structure for users.

Related controls may also be associated with ARIA. ARIA provides a grouping role that functions similarly to the fieldset and legend. role="group" with an aria-labelledby pointing to a visible text node is a sufficient technique as well.

Designing accessible grouping controls in forms is essential for ensuring all users can interact with content effectively. Follow these guidelines to create accessible form designs in Figma:

  1. Provide Legends and Labels: Use legends for fieldsets and labels for form controls. Legends can be visible or hidden but should clearly describe the group of form controls.
  2. Ensure Proximity of Labels to Controls: Place labels in close proximity to their corresponding form controls to ensure the relationship is clear.
  3. Design Visual Cues for Required Fields: Clearly indicate which fields are required using visual cues, such as asterisks (*) next to the labels of required fields. Include instructions at the beginning of the form explaining what the cues mean.
  4. Ensure Sufficient Color Contrast: Design form elements with sufficient contrast between text labels, input text, and background colors to meet or exceed the minimum contrast ratios defined by WCAG guidelines.
  5. Design for Different States: Include designs for various form states, such as hover, focus, and success states, to provide feedback to users as they interact with the form.
  6. Consider the Order of Form Elements: Organize form elements in a logical order that follows natural reading patterns, typically top to bottom and left to right, to ensure a smooth flow for users filling out the form.

Developing accessible forms is essential for compliance with WCAG standards and ensuring usability for all users. Utilize the design and audit annotations to guide your development:

  1. Group Related Form Controls with Fieldset: Use the <fieldset> element to group related form controls. This creates a visual and structural grouping, making it easier for users to understand the relationship between the controls.
  2. Provide a Legend for Fieldset: Use the <legend> element within the <fieldset> to provide a caption or label for the group. This label helps users understand the purpose of the grouped controls.
  3. Use ARIA Grouping Role: In cases where <fieldset> and <legend> are not suitable (e.g., non-form elements), use role="group" with the aria-labelledby attribute. Ensure the aria-labelledby attribute references an element that provides a descriptive label for the group.
  4. Ensure Accessibility of Group Labels: Make sure that the text within <legend> or the element referenced by aria-labelledby is descriptive and provides clear context about the grouped controls.
  5. Maintain Keyboard Navigation: Ensure that all grouped controls can be accessed and navigated using a keyboard. Grouping should not interfere with the logical tab order or keyboard operability of individual controls.
  6. Consistent Design and Markup: Apply consistent design and markup for grouping controls across your forms. This includes using similar styles and structures for all fieldsets or ARIA groups to maintain a cohesive user experience.
