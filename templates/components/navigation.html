When a navigation bar has a disclosure button revealing a sub menu these accessibility considerations must also be included:

Submenu Disclosure Button

  • Must be a <button> OR have role="button" and tabindex="0" and function equally with mouse and keyboard. (May need additional event handlers added)
  • Must have toggling aria-expanded="true/false" depending on expanded/collapsed states.
  • Should have aria-controls attribute equal to the ID of the container of the submenu it controls.

Submenu Disclosure Panel

  • List of links must be in list markup
  • Each link must have a valid href, and the text content/name of the control should give the link purpose and reflect its destination.
  • The panel must collapse when focus leaves with forward and backwards tabbing navigation so it does not obscure of content.

Mobile Navigation with Disclosure Button Available on Resize/Reflow

To see an example of Moble Navigation with a 'Hamburger' disclosure button you may resize this page smaller than 1000px. Once resized you will see the simple nav bar examples transform into a hidden panels with visibility that is controled by Hamburger disclosure buttons.

The mobile disclosure must:

  • Have an accessible name for the hamburger button given with an aria-label, aria-labelledby or sr-only text.
  • The hamburger button must have a toggling aria-expanded="true\false" that accurately reflects the expanded/collapsed state.
  • The hamburger button must also have the attribute aria-controls referencing the unique ID of the panel it controls.
  • The interactive elements within the panel must be in list markup and must be properly hidden with a display:none; or visibility:hidden; property that ensures they are not in reading or focus order when they are not visible.

  • Consider Color Contrast: Design with high contrast ratios for text, graphical objects, and button borders to ensure readability and visibility for users with visual impairments. Please reference General Color section for more detials.
  • Visual Indicators for Disclosure Buttons: Include a visual indicator, such as a caret, for disclosure buttons to signify their function of opening submenus, aiding in setting the right user expectations.
  • Differentiate Submenu Buttons from Links: Clearly distinguish submenu buttons from hyperlinks in their design to accurately reflect their functionality, focusing on visual differentiation in the user interface.
  • Interactive Element Cues: Provide intuitive visual cues for elements within the navigation bar to clearly convey their functionality, ensuring users understand how to interact with each element.
  • Legibility of Text and Icons: Design text and icons to be legible and scalable, maintaining readability across different screen sizes and resolutions.
  • Consistent Navigation Design: Keep the navigation bar design consistent across different pages and states to aid user familiarity and improve overall navigation usability.

  • Color Contrast: Assess the navigation bar to ensure that text, icons, and graphical elements have sufficient color contrast against their background colors. This is to guarantee readability and visibility for users with visual impairments.
  • Visual Indicators for Disclosure Buttons: Review the design for clear visual indicators on disclosure buttons or dropdowns, such as carets, to signify their expandable nature. Lack of clear indicators should be noted for design improvements.
  • Differentiation Between Submenu Buttons and Links: Check that submenu buttons and links are distinctly designed to accurately reflect their functionality. Ensure that the design visually differentiates between clickable links and buttons that trigger submenus.
  • Interactive Element Cues: Inspect interactive elements within the navigation bar for intuitive visual cues that clearly convey their functionality. This includes examining hover states, selected states, and other visual feedback mechanisms.
  • Legibility of Text and Icons: Evaluate the legibility of text and the clarity of icons within the navigation bar, especially considering various screen sizes and resolutions. Flag any elements that compromise legibility or scalability.
  • Consistency in Navigation Design: Ensure that the navigation bar's design is consistent across different pages and states. Inconsistencies in design can lead to confusion and should be highlighted for correction.

  • Semantic HTML: Use semantic HTML elements to structure the navigation bar. This includes using the <nav> element to wrap the entire navigation bar and <ul>, <li>, and <a> for navigation items to ensure proper screen reader interpretation.
  • Roles and Properties: Apply appropriate ARIA roles and properties where native HTML does not suffice. This is particularly important for complex widgets like disclosure buttons, which should have aria-expanded, aria-controls, and role="button" attributes to communicate state and functionality.
  • Keyboard Navigation: Ensure all interactive elements within the navigation bar are operable with keyboard alone. This includes ensuring tab order is logical and that submenu items are accessible via keyboard commands.
  • Focus Management: Manage focus for interactive components, particularly for navigation bars with submenus. When a submenu is opened via a disclosure button, focus should move to the first item in the submenu. On close, focus should return to the disclosure button.
  • Color Contrast: Implement sufficient color contrast ratios for text, icons, and interactive elements against their background colors. Please see General Color section for more details.
  • Visible Focus Indicators: Provide clear visual focus indicators for interactive elements to help users understand which element has focus during keyboard navigation.
  • Testing with Screen Readers: Test the navigation bar with screen readers to ensure that all text is read correctly, and that interactive elements are announced with their state (e.g., expanded or collapsed for submenus).
  • Consistent Navigation: Keep the navigation structure consistent across the entire website to help users predict where navigation elements will be located and how they will function, enhancing usability and accessibility.
