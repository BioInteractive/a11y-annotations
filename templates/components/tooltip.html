Tooltip

A tooltip is a pop-up element that provides additional information about a button, link, form field, or any focusable element. It becomes visible either when the user hovers over the element with a mouse or when it gains focus through the keyboard, typically appearing after a brief delay. The tooltip remains visible as long as it is focused or hovered over and disappears when the user presses the Escape key or moves the mouse away. Importantly, the focus remains on the triggering element, not on the tooltip itself.

  • Tooltip Display: Tooltip must display on both hover and focus events, ensuring that it is operable through keyboard interfaces as well as by mouse. 2.1.1 Keyboard
  • Text Association: Tooltip text must be associated with the UI control it describes using aria-describedby. This ensures users of assistive technologies understand the relationship and context. 1.3.1 Info and Relationships
  • Dismissible: Tooltip must be dismissible with the ESC key, especially if it covers other content, to provide a way to close the tooltip without moving the mouse or changing focus. 1.4.13 Content on Hover or Focus
  • Persistent Visibility: Tooltip text must remain visible as long as the triggering element has focus or the mouse hovers over it, to ensure it is usable by people who need longer to read it. 1.4.13 Content on Hover or Focus

Hidden text must be visually hidden AND programmatically hidden from assistive technologies to preserve the correct reading order with aria-hidden="true".

Authors SHOULD ensure that elements with the role tooltip are referenced through the use of aria-describedby before or at the time the tooltip is displayed. The role="tooltip" is not widely supported by assistive technologies.

Role, Property, State, and Tabindex Attributes

Role Attribute Element Usage
tooltip span Identifies the element as a tooltip, providing additional descriptive information.
aria-describedby="tooltiptext" button Associates the tooltip with the control element for which it provides additional information.
aria-hidden="true" span Ensures the tooltip is not visible to assistive technologies unless it is activated.

  • Trigger Clarity: Design tooltips to be clearly indicated by the interface elements they are associated with. Use visual cues like underlines or help icons next to form labels or buttons to signify that additional information is available.
  • Contrast and Visibility: Ensure that the tooltip text and background have a high color contrast ratio (at least 4.5:1) to be easily readable by users with visual impairments. The tooltip design should stand out against the rest of the application content.
  • Size and Positioning: Design tooltips to appear close to their trigger elements without obstructing other user interface elements or important information. Ensure tooltips do not change layout significantly, which could disorient users.
  • Content Brevity: Keep the tooltip content concise and to the point. Tooltips are meant for brief snippets of supplemental information, not detailed explanations.
  • Non-reliance on Color: Do not rely solely on color to convey the tooltip's message. Text within the tooltip should clearly describe the information or instruction without requiring color perception for understanding.

When auditing a Tooltip component within a Figma design, it is crucial to review the documentation provided by content leads and the implementation by designers for consistency and alignment with accessibility standards.

During the audit of the Tooltip component in Figma, consider the following accessibility aspects to flag any issues for the designer's review:

  • Tooltip Position: Confirm that the tooltip is visible in proximaty to the triggering control. These must have a visual relationship by proximity.
  • Text Legibility: Review the tooltip text for sufficient color contrast against its background to ensure legibility for users with visual impairments.
  • Content Clarity: Examine the clarity and conciseness of the tooltip text, ensuring it is free from jargon and easily understandable, providing valuable information succinctly.

Tooltip

  • Tooltip must display on hover & focus
  • Tooltip must be dismissible with ESC key - if it covers any other content on any screen size.
  • Tooltip text must be associated with UI control. Use aria-describedby to associate the tooltip with the UI control.
  • Tooltip text must be persistently visible while mouse hovers over tooltip text.

  • Focusable Triggering Element: Ensure the element that triggers the tooltip is focusable and appears in the document's focus order. The tooltip must activate automatically when the element receives focus.
  • Mouse Interaction: The tooltip should also appear when the user hovers over the triggering element with a mouse. Ensure the tooltip remains visible as long as the mouse hovers over the element or the tooltip itself, and disappears when the mouse leaves both.
  • Dismissible: Tooltips covering other content must be dismissible using the ESC key. Implement this functionality to enhance usability and ensure compliance with accessibility standards.
  • ARIA Association: Use aria-describedby to link the tooltip with its triggering control, providing assistive technologies with contextual information about the tooltip's purpose. ARIA: aria-describedby
  • Managing Visibility with ARIA: Use aria-hidden to control the visibility of the tooltip to assistive technologies. Set aria-hidden="true" when the tooltip is not visible and toggle it as the tooltip shows or hides.
  • Text Legibility and Contrast: Ensure the text within the tooltip has sufficient contrast against its background to comply with WCAG 2.1 1.4.3 Contrast (Minimum). This guarantees readability for users with visual impairments.
