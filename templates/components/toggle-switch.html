Toggle Switch Button

A toggle switch is an interactive element that allows users to switch between two states, typically "on" and "off." This component must be accessible to all users, including those using keyboard navigation and assistive technologies. Below are the WCAG success criteria requirements specific to toggle switches, ensuring they are usable and understandable for everyone.

  • Keyboard Operability: Toggle switches must be operable through both keyboard and mouse interfaces. Users should be able to toggle the switch using the Enter or Space keys. 2.1.1 Keyboard
  • Accessible Name: Toggle switches must have an accessible name that clearly describes their function. This can be achieved using aria-labelledby or aria-label. 4.1.2 Name, Role, Value
  • Focus Indicator: Ensure that the toggle switch provides a visible focus indicator when it receives focus, making it easy for keyboard users to see where they are. 2.4.7 Focus Visible
  • State Information: The current state of the toggle switch (e.g., checked or unchecked) should be conveyed through the aria-checked attribute. Ensure that the state is perceivable by assistive technologies. 1.3.1 Info and Relationships.

Use distinct visual cues, such as color changes and text labels within the switch, to show the current state. Ensure that these states are easily distinguishable by users with visual impairments. In Figma, use layers and styles to manage these states clearly.
  • Color Contrast: Ensure that the toggle switch and its state indicators have a high color contrast ratio (at least 4.5:1) against the background. Utilize Figma plugins like "Contrast" to check and ensure adequate color contrast.
  • Focus Indicators: Provide a clear visual focus indicator for the toggle switch when it receives keyboard focus. In Figma, use styles and effects to create focus states, such as adding a border or shadow.
  • Target Size: Design the toggle switch with a large enough target area to be easily activated. The target size should be at least 44x44 pixels to accommodate users with motor impairments or those using touch devices. Use Figma’s frame and layout tools to ensure proper sizing.
  • Text Labels: Include text labels inside or next to the toggle switch to describe the "on" and "off" states. Use clear, concise language for these labels to ensure they are easily understood. In Figma, ensure text layers are properly named and organized.
  • Non-reliance on Color Alone: Do not rely solely on color to indicate the state of the toggle switch. Ensure that the state is also conveyed through text or other visual indicators to accommodate users who cannot perceive color differences. Use symbols and icons in Figma to provide additional cues.
  • Hover and Pressed States: Design different visual states for the toggle switch when it is hovered over or pressed. This provides feedback to users about their interaction with the element. Use Figma’s interactive components feature to create these states and transitions.
  • Alignment and Placement: Place the toggle switch in a consistent and logical location within the user interface. It should be aligned with related form elements or controls to ensure a clear and intuitive user experience. Use Figma’s alignment and distribution tools to maintain consistency.
  • Responsive Design: Ensure the toggle switch is responsive and works well across different screen sizes and devices. Use Figma’s auto-layout feature to create components that adjust to different screen sizes.

When auditing a toggle switch component within a Figma design, it is crucial to review the documentation provided by content leads and the implementation by designers for consistency and alignment with accessibility standards.

During the audit of the toggle switch component in Figma, consider the following accessibility aspects to flag any issues for the designer's review:

  • Switch State Visibility: Confirm that the current state of the toggle switch (e.g., on/off) is visually clear and distinguishable. Use color changes and text labels to indicate state.
  • Color Contrast: Review the toggle switch and its state indicators for sufficient color contrast against the background to ensure visibility for users with visual impairments.
  • Focus Indicators: Check that the toggle switch has a clear and visible focus indicator when navigated to via keyboard to assist users who rely on keyboard navigation.

Toggle Switch

  • Must be operable with keyboard (Enter and Space keys) as well as mouse and touch.
  • Must have clear on/off state indications with text and/or icons.
  • Must maintain a visible focus indicator when focused via keyboard navigation.
  • Must have sufficient color contrast (at least 4.5:1) for all states against the background.
  • Must have a minimum target size of 44x44 pixels.

  • Keyboard Operability: Ensure the toggle switch can be operated using the Enter and Space keys. This allows users who rely on keyboard navigation to interact with the switch effectively.
  • Accessible Name: The toggle switch must have an accessible name that clearly describes its function. Use aria-labelledby or aria-label to provide this information.
  • State Information: Use the aria-checked attribute to indicate the current state of the toggle switch. Set aria-checked="true" when the switch is on, and aria-checked="false" when it is off. Ensure this state information is perceivable by assistive technologies.
  • Focus Indicator: Ensure the toggle switch provides a visible focus indicator when it receives focus. This helps keyboard users to see which element is currently focused.
  • Color Contrast: Verify that the toggle switch and its state indicators have sufficient color contrast (at least 4.5:1) against the background to ensure visibility and readability for users with visual impairments.
  • Target Size: Design the toggle switch with a minimum target size of 44x44 pixels to accommodate users with motor impairments or those using touch devices.
  • Non-reliance on Color Alone: Do not rely solely on color to indicate the state of the toggle switch. Use text or other visual indicators to convey the state for users who cannot perceive color differences.
  • Hover and Pressed States: Implement different visual states for the toggle switch when it is hovered over or pressed. This provides feedback to users about their interaction with the element.
  • ARIA Roles and Properties: Properly implement ARIA roles and properties to ensure that the toggle switch is recognized and operable by assistive technologies. The switch should have the role of switch and use aria-checked to reflect its state.
