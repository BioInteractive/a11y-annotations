Alert

  • Proper Use of ARIA Roles and Properties: Implement role="alert" on alert components to ensure they are recognized as alerts by assistive technologies, automatically announcing changes without needing focus. ARIA: Alert Role
  • Automatic Processing: Alerts should be processed as atomic live regions with aria-live="assertive" to interrupt other activities and read the message immediately. 4.1.3 Status Messages
  • Text Contrast: Ensure that text within alerts has sufficient color contrast to meet accessibility guidelines, making it readable against its background. 1.4.3 Contrast (Minimum)
  • Screen Reader Accessibility: Use aria-atomic="true" on alerts to ensure that the entire message is read as a whole rather than part of it when updates occur. ARIA: aria-atomic
  • Visual Design: Design alerts to stand out visually from other content on the page to ensure they capture the user's attention immediately. 1.4.11 Non-text Contrast

Role, Property, State, and Tabindex Attributes

Role Attribute Element Usage
alert div Identifies the element as the container where alert content will be added or updated.
aria-live="assertive" Implicit on div
  • This does not have to be declared in the code because it is implicit in the alert role.
  • Tells assistive technologies to interrupt other processes to provide users with immediate notification of relevant alert container changes.
aria-atomic="true" Implicit on div
  • This does not have to be declared in the code because it is implicit in the alert role.
  • Tells assistive technologies to use the entire content of the alert element as the alert message even if only a portion of it has changed.

  • Visual Clarity: Ensure alerts are highly noticeable with visual cues such as bold colors, distinctive borders, or varied shapes to distinguish them clearly from other content.
  • Use of Icons: Incorporate icons that indicate the type of alert (e.g., error, warning, success). Select icons that are intuitive and universally recognizable to enhance quick comprehension.
  • Color and Icon Contrast: Apply high color contrast for text and icons within alerts against their backgrounds to ensure legibility for all users, especially those with visual impairments. Icons used must also meet contrast requirements to be distinguishable.
  • Consistent Design: Maintain consistent design elements for alerts across the application or website. Consistency in icon use, color schemes, and placement helps users quickly recognize and understand alerts whenever they appear.
  • Avoiding Color Reliance: While color enhances visual communication, it should not be the sole method for conveying the nature of the alert. Ensure that information conveyed by color is also available through text or icons to accommodate users who cannot perceive color differences.

When auditing and an Alert component please review the content leads documentation and the designer's implementation for alignment and consistency. Then review for accessibility considerations to flag for the designer and then annotate the component requirements for developer handoff.

When auditing the Alert component within a Figma design please consider the following accessibility issues to flag for the designer to review:

  • Color Contrast of Text: Verify that the text within the alert has sufficient color contrast against its background, meeting the minimum contrast ratio requirement of 4.5:1, to ensure legibility for users with visual impairments.
  • Color Contrast of Icons: Check that any icons used within the alert also meet the required contrast ratio of 3:1 against their background to ensure they are easily distinguishable.
  • Proximity to Referenced Elements: Ensure that alerts are positioned close to the elements they relate to, or are clearly associated, so users can easily understand what the alert is referencing.
  • Clarity of Message: Review the language used in the alert to ensure it is clear, concise, and easily understandable. Avoid jargon or overly technical terms that could confuse users.
  • Consistent Icon Usage: Confirm that icons are used consistently across similar types of alerts to help users quickly identify the alert type based on previous interactions.
  • Avoidance of Color-Only Information: Check that alerts do not rely solely on color to convey information. Ensure that all information conveyed through color is also available in text or iconography.

Alert

Must be announced and associated where applicable.

  • Add role="alert" to alert container; ensure it is in the DOM on page load to function as a live region.
  • Inject alert messages dynamically into the live region using JavaScript to ensure screen readers announce changes immediately.
  • Associate alert messages related to specific elements using aria-describedby linked to the unique ID of the alert container.
  • Keep alerts visible until dismissed by the user or the related issue is resolved; include an accessible dismiss mechanism if applicable.
  • Ensure all interactive elements within alerts, like links or buttons, are keyboard accessible and meet color contrast requirements.
  • Hide decorative icons from assistive technologies with aria-hidden="true"; provide alt text or labels for informative icons.

  • Role and Live Region: Apply the role="alert" to the container element that will house the alert message. Ensure this element is in the DOM on page load so it is recognized as a live region by assistive technologies. ARIA: Alert Role
  • Dynamic Content Injection: Use JavaScript to dynamically inject the alert message into the designated live region. This ensures the message is announced by screen readers immediately upon changes or event triggers.
  • Association with Interactive Elements: If the alert is related to a specific interactive element (like a form field), use aria-describedby to associate the alert with the element by linking to the unique ID of the alert container. This provides contextual information to users about what the alert is referring to. ARIA: aria-describedby
  • Persistent Visibility: Ensure the alert message remains visible until the user dismisses it or the issue prompting the alert is resolved. If a dismiss mechanism is provided (e.g., a close button), ensure it is accessible and clearly marked with an accessible name using aria-label.
  • Accessibility of Interactive Elements: For alerts that include interactive elements like links or buttons, ensure these are fully accessible, with keyboard support and sufficient color contrast.
  • Hiding Decorative Icons: If icons used in the alert are decorative and do not convey additional information, hide them from assistive technologies using aria-hidden="true". If they do convey information, ensure they have appropriate alt text or labels.
