Non-text Color Contrast

The visual presentation of the following have a contrast ratio of at least 3:1 against adjacent color(s):

  • User Interface Components
    Visual information required to identify user interface components and states, except for inactive components or where the appearance of the component is determined by the user agent and not modified by the author;
  • Graphical Objects
    Parts of graphics required to understand the content, except when a particular presentation of graphics is essential to the information being conveyed.

  1. Choose a color contrast measuring tool that allows you to sample colors from your webpage. See Resources below for a recommended list of tools.
  2. Determine if the UI component or Graphical Object is meaningful and required to identify information or states.
  3. In the color contrast tool, you'll typically find color droppers or eyedropper tools. These tools allow you to select specific colors from the design.
  4. Use color dropper tool to select foreground element (e.g., object) and then select background element (e.g., the background behind the object)
  5. Once you've sampled both foreground and background colors, the color contrast tool should display the contrast ratio between the two colors. This contrast ratio is typically calculated as the ratio between the luminance (brightness) of the foreground and background colors.
  6. Compare the contrast ratio against the WCAG requirements of a 3:1 minimum ratio to determine a pass or fail.

  • Border of the interactive element or graphical object must have 3:1 contrast with the background.
  • Warning: the contrast ratio fails the requirement with a current ratio of ____.
  • Recommendation: Adjust color combination to meet the minimum of 3:1 contrast ratio requirements.
