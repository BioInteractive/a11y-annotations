  • Must have a unique name if multiple navigational elements are present on the same page. 4.1.2 Name, Role, Value

Simple Breadcrumb Example

  1. Home
  2. Library
  3. Data

Current page link must have aria-current="page" so the current state is programmatically determinable. If there is a visual indication of state, such as color then the current and not current links must have a minimum of 3:1 contrast ratio. A visual indication is not a requirement because the visual hierarchy is considered sufficient visual indication.

  • Visual Clarity and Hierarchy: Design breadcrumb items with clear, legible typography and ensure adequate spacing for easy identification and interaction.
  • Color Contrast: Ensure text and interactive elements within the breadcrumb meet WCAG contrast ratio guidelines to guarantee readability for users with visual impairments.
  • Consistent Design and Placement: Maintain a consistent design and placement for the breadcrumb across different pages to aid in user navigation and orientation.
  • Current Page Indication: If you add a visual indicator that separates it from other links. Also, include consideration for a contrast ratio of at least 3:1 with other breadcrumb items to highlight its status or use a non color indicator such as bold text or an underline.
  • Use of Separators: Incorporate visually clear separators between breadcrumb items. These should be designed to be recognized as decorative and non-interactive.
  • Interactive Feedback: Provide hover and focus states for breadcrumb links to offer visual feedback, aiding users in understanding which items are interactive.

  • Color Contrast for Text: Test the color contrast of regular-sized text in breadcrumb links against their background to ensure it meets a minimum of 4.5:1 ratio, aligning with WCAG 2.1 guideline 1.4.3 Contrast (Minimum).
  • Color Contrast between Current and Not Current Links: If a visual indication of state (e.g., color differentiation) is used for the current page link, verify that the contrast ratio between the current page link and not current links is at least 3:1. This ensures that users can distinguish the current page within the breadcrumb trail, in compliance with WCAG 2.1 guideline 1.4.3 Contrast (Minimum).
  • Visual Hierarchy as Indication: Acknowledge that a separate visual indication for the current page link is not a strict requirement, provided that the breadcrumb's design leverages visual hierarchy effectively to indicate the current page's status.

To annotate for any text or text state color failures please visit the color section under the general tab.

Breadcrumb Navigation

  • Must be contained in a <nav> element OR a sectioning element such as a <div> with role="navigation"
  • If multiple navigation elements are present on a page, such as a Primary and a Breadcrumb navigation component, each must provide additional labeling for screen reader users. Add aria-label="Breadcrumb"
  • If current page navigation link has visual indicator then it must also be programmatically determinable with aria-current="page". The visual indicator must have a minimum 3:1 contrast ratio with the adjacent colors and/or another non-color distinguishing visual indicator, such as bold text, an underline, a border, etc.
  • Breadcrumb navigation text and links must be in list markup <ul> and links must be keyboard navigable in <a> elements with valid href's and link purpose.

  • Use Semantic Navigation Landmarks: Wrap the breadcrumb component in a <nav> element or use a sectioning element with role="navigation" to ensure it's recognized as a navigation landmark, aiding assistive technologies in understanding its purpose.
  • Ensure Unique Names for Multiple Navigation Landmarks: If your page includes multiple navigation landmarks, assign unique names to each (using aria-label or aria-labelledby) to distinguish them for users of assistive technologies.
  • Maintain Consistency Across Pages: Ensure the breadcrumb's structure and presentation remain consistent across different sections of the site, facilitating user navigation and orientation.
  • Structure Breadcrumb Items Properly: Organize breadcrumb links within an ordered list (<ol>) to reflect the sequential nature of navigation paths, using <li> for each breadcrumb item to ensure semantic structure.
  • Indicate the Current Page with ARIA: Apply aria-current="page" to the last breadcrumb link, which represents the current page. This attribute offers a clear, programmatic indication of the breadcrumb's active state to assistive technologies.
  • Color Contrast Compliance: Verify that the text of breadcrumb links, including the current page indicator, meets the minimum color contrast ratio of 4.5:1 against their background for regular text, in accordance with WCAG 2.1 guideline 1.4.3 Contrast (Minimum). Additionally, ensure a minimum contrast ratio of 3:1 between the current page link and other links if a visual state indication is utilized.
  • Hide Decorative Separators from Assistive Technologies: If using decorative separators (e.g., slashes or arrows) between breadcrumb items, use aria-hidden="true" to hide these elements from screen readers, as they do not convey meaningful content.
  • Option Text for Current: It is absolutely acceptable to use text instead of a link for the current breadcrumb item.
