Disabled links must have tabindex="-1" to ensure they are not in focus order and they must have the aria-disabled="true" attribute so the link is still detectable to assistive technologies but alerts users is not functional.

Active links must have a visual indication of state that has a minimum of 3:1 contrast ratio in inactive links and must have aria-current="page" To ensure the link's current state is programmatically determinable to assitive technologies.

  • Visual Clarity: Design pagination elements (e.g., page numbers, 'Previous', 'Next' buttons) with clear, legible typography and sufficient spacing to ensure they are easily identifiable and clickable.
  • Color Contrast: Ensure that all pagination elements have a high contrast ratio against their background, meeting WCAG guidelines, to guarantee readability by users with visual impairments.
  • Disabled State Visuals: Design a distinct style for disabled pagination elements (e.g., 'Previous' button on the first page) that indicates they are not interactive, while ensuring they are still visually recognizable.
  • Active State Visuals: Create a unique design for the active page number that distinguishes it from other page numbers, using color, font weight, or other visual indicators to signify the current page clearly.
  • Consistent Navigation Design: Maintain consistency in pagination design across different parts of the website to help users become familiar with navigating paginated content.
  • Icons and Indicators: If using icons (e.g., arrows for 'Previous' and 'Next'), ensure they are simple, recognizable, and supplemented with text labels or visually implied meanings for clarity.
  • Hover and Focus States: Design hover and focus states for interactive elements in the pagination to provide visual feedback when users interact with them, even though actual implementation is developer-dependent.
  • Visual Indicators for Large Pagination: If designing for a large set of pages, consider visual cues (e.g., ellipses) to indicate skipped sections of pages and design these elements for clear visibility and understanding.

  • Color Contrast: Check that text and visual elements of the pagination component have sufficient color contrast ratios as per WCAG guidelines to ensure legibility for users with visual impairments.
  • Visual Clarity of Interactive Elements: Verify that interactive elements such as page numbers, 'Previous', and 'Next' buttons are visually distinguishable from non-interactive elements and clearly indicate their purpose.
  • Disabled State Indication: Review designs for disabled states (e.g., 'Previous' button on the first page) to ensure they are clearly indicated as non-interactive while remaining visible to the user.
  • Active State Indication: Ensure that the current page number is visually distinct from other page numbers to clearly indicate the current position in the pagination sequence.
  • Consistency Across Pages: Assess the consistency of pagination design across different sections or pages of the product to ensure a predictable navigation experience for users.
  • Icons and Visual Indicators: Examine the use of icons or visual indicators (like arrows or ellipses) for their simplicity, recognizability, and whether they are accompanied by clear, descriptive labels or implied visual meanings.
  • Hover and Focus State Designs: Check for the presence of hover and focus state designs for interactive elements, recognizing that actual state changes will be verified in development.
  • Design for Large Sets of Pages: For designs accommodating large sets of pages, ensure visual indicators for page skipping (e.g., ellipses) are clear and understandable.

Pagination

  • Must be in a navigation landmark: Use <nav> or a sectioning element with role="navigation".
  • Must have a unique name if multiple navigational elements are present on the same page.
  • Must be consistent across all pages of a website and ordered consistently.
  • If the navigation element contains a list of links, those must be in a list structure with valid link markup.
  • Any disabled links must have aria-disabled="true" and tab-index="-1".
  • Any active links with visual indication must also have aria-current="page".

  • Use Semantic Navigation Landmarks: Wrap the pagination component in a <nav> element or use a sectioning element with role="navigation" to ensure it's recognized as a navigation landmark.
  • Ensure Unique Names for Multiple Navigation Landmarks: If your page contains multiple navigation landmarks, make sure each has a unique name to distinguish them for assistive technologies.
  • Maintain Consistency Across Pages: The order and presentation of pagination links should be consistent across all pages of the website, aiding in user orientation and predictability.
  • Structure Pagination Links Properly: Organize pagination links within a list (<ul> or <ol>) to ensure a clear structure, using <li> for each pagination link for semantic accuracy.
  • Mark Disabled Links Appropriately: For any non-clickable or disabled pagination links (e.g., the current page or out-of-bound links), use aria-disabled="true" and ensure they're not focusable with tabindex="-1".
  • Indicate the Current Page with ARIA: Use aria-current="page" on the pagination link that represents the current page, providing a programmatic indication of the active state.
