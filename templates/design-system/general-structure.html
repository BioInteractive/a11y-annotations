Design System General Structure

Structural Foundations of the BioInteractive Design System

The BioInteractive Design System establishes clear structural rules for organizing content across interactive experiences. These rules define how page titles, headings, text, spacing, and lists should be used to create readable, predictable, and accessible layouts.

By standardizing these foundational elements, designers and developers can ensure consistent user experiences across projects while supporting accessibility requirements from the start.

Page Titles

Page titles provide users with a clear and descriptive summary of a page’s purpose. Every page should begin with a single primary heading that communicates what the page contains and how it relates to the overall experience.

Titles should be concise, descriptive, and aligned with the content that follows. Avoid vague titles such as “Home” or “Untitled.” Instead, titles should clearly describe the content or activity presented on the page.

A consistent page title structure helps users orient themselves within complex educational content and supports screen reader navigation.

Heading Hierarchy

Headings provide the structural framework for page content. They allow users to quickly scan content and help assistive technologies communicate the relationships between sections.

The BioInteractive design system uses a clear heading hierarchy from H1 through H6. Headings should always be used in a logical order to maintain a meaningful document structure.

  • Each page should contain a single H1 representing the page title.
  • Major sections should use H2.
  • Subsections should use H3 and lower levels when needed.
  • Heading levels should never be skipped.

Heading Scale and Responsive Typography

The design system defines a consistent heading scale to maintain visual hierarchy across both desktop and mobile layouts. Each heading level has defined font sizes and line heights that adapt to different screen sizes.

Maintaining clear visual differences between heading levels ensures that users can quickly understand the structure of the content.

Responsive typography ensures that headings remain readable and visually balanced regardless of screen size.

Text Readability

Readable text is essential for accessibility and comprehension. The BioInteractive design system encourages typography choices that support a wide range of users, including people with cognitive disabilities and non-native readers.

  • Use easily readable typefaces.
  • Maintain sufficient color contrast between text and background.
  • Use appropriate font sizes for body content.
  • Provide adequate line spacing to improve readability.
  • Avoid fully justified text alignment.

Body Text Standards

Body text should prioritize readability across devices and screen sizes. The design system defines recommended font sizes and line heights to ensure content remains easy to read.

Paragraph text should maintain comfortable spacing between lines and paragraphs to support sustained reading. Designers should avoid overly dense text blocks that may create visual fatigue.

Lists

Lists help organize related information and improve scannability. When information functions as a list, it should be structured using proper list formatting rather than relying on manual styling.

  • Use unordered lists for grouped information without a required order.
  • Use ordered lists when sequence or steps are important.
  • Use consistent bullet styles to maintain visual clarity.

Proper list structure also ensures that screen readers communicate grouped content correctly.

Spacing and Layout Rhythm

Spacing plays a critical role in readability and visual hierarchy. The design system establishes consistent spacing between headings, paragraphs, buttons, and lists to maintain visual rhythm.

Standard spacing helps users visually distinguish between content groups and improves the overall clarity of the interface.

  • Maintain consistent spacing between headings and paragraphs.
  • Provide adequate spacing between paragraphs and interactive elements.
  • Use whitespace to separate content groups without introducing unnecessary clutter.

Connecting Structure to Accessibility

The structural rules defined in the BioInteractive design system support accessible navigation and predictable interaction. Proper use of headings, lists, spacing, and readable typography ensures that content can be understood by both visual users and assistive technologies.

Designers should apply these structural guidelines consistently to support accessible learning experiences across all BioInteractive projects.
