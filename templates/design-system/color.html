Design System Color

Color System Overview

The BioInteractive color system defines the brand palette and interface colors used across digital products and interactive learning experiences. These colors provide visual consistency while supporting accessibility requirements for contrast, readability, and usability.

Color choices in the system are intentionally limited and structured so designers can create clear visual hierarchy without introducing contrast failures or confusing visual signals.

Primary Brand Colors

The BioInteractive design system uses a defined set of primary brand colors that represent the organization’s visual identity. These colors should be used consistently across interfaces to reinforce brand recognition.

The primary palette includes the core HHMI teal and green tones along with neutral colors used for text and backgrounds.

  • Teal and green tones represent the BioInteractive brand identity.
  • Neutral colors such as black, gray, and white provide contrast and structure.
  • Red is reserved for warning or error states.

Secondary Brand Colors

Secondary colors provide additional flexibility when designing interactive content. These colors should be used sparingly and should never replace the primary brand palette.

Secondary colors can help distinguish sections of content, highlight educational elements, or support visual differentiation within interactive experiences.

Website and Interface Colors

The design system defines a set of colors specifically intended for use in the user interface. These colors are used for text, backgrounds, buttons, icons, borders, and interactive elements.

  • Dark text colors are used for body copy and headings.
  • Light background colors support readability and contrast.
  • Accent colors may be used for buttons, icons, and emphasis.

Using a limited set of interface colors helps maintain visual clarity and ensures that accessibility standards are easier to maintain across the system.

Use of Color

Color should support meaning and visual hierarchy, but it should never be the only method used to communicate information. Users with color vision deficiencies may not be able to distinguish certain color differences.

  • Do not rely on color alone to convey information.
  • Use icons, labels, patterns, or text to reinforce meaning.
  • Ensure interactive states remain distinguishable beyond color.

Providing multiple visual cues ensures that all users can understand and interact with the interface.

Text Color Contrast

Text contrast between foreground and background colors is essential for readability. The design system follows WCAG contrast requirements to ensure text remains readable for users with low vision and other visual impairments.

  • Body text should maintain a contrast ratio of at least 4.5:1.
  • Large text may use a minimum contrast ratio of 3:1.
  • Text must remain readable across all interface backgrounds.

Designers should test color combinations to ensure that text remains legible under different lighting conditions and device displays.

Non-Text Contrast

Non-text elements such as icons, buttons, form controls, and graphical indicators must also meet accessibility contrast requirements.

  • Interactive components should maintain at least a 3:1 contrast ratio against adjacent colors.
  • Icons must remain distinguishable against their background.
  • Buttons and interactive states must remain visually clear.

Maintaining sufficient non-text contrast ensures that users can perceive interface elements and interact with them confidently.

Color Contrast Examples

The design system includes visual examples showing acceptable and unacceptable color combinations for icons, buttons, and text. These examples demonstrate how color contrast should be applied in real interface scenarios.

Designers should use these examples as reference when creating new components or selecting colors within the interface.

Designing with Accessibility in Mind

Accessible color usage improves usability for everyone. When selecting colors, designers should prioritize readability, clarity, and consistent meaning across the interface.

Following the color guidelines in this design system helps ensure that BioInteractive experiences remain accessible, visually cohesive, and easy to understand for all users.
