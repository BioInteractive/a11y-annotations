Design System Components

Component Library

The BioInteractive design system includes a library of reusable interface components that support interactive learning experiences. These components provide consistent patterns for interaction, feedback, and navigation across the platform.

Using standardized components helps designers and developers maintain predictable behavior, visual consistency, and accessibility across projects. Each component links to a detailed documentation page within the Accessibility Annotation Repository.

Interface Components

Basic interface components provide the foundation for many interactions throughout the system.

  • Links allow users to navigate between pages or resources.
  • Buttons trigger actions such as submitting forms or opening dialogs.
  • Accordions allow sections of content to expand and collapse.
  • Navigation, Breadcrumbs, and Pagination support structured movement through content.

Interactive Components

Interactive components provide contextual information or allow users to control interface behavior.

  • Tooltips provide short contextual explanations.
  • Toggletips display additional supporting information.
  • Toggle switches allow users to enable or disable settings.
  • Carousels present rotating content.
  • Video components support multimedia learning content.

Data and Interface Controls

These components help users explore information, interact with data, or navigate complex interfaces.

  • Sliders with input allow users to adjust numeric values.
  • Tables organize structured information.
  • Tabs divide content into manageable sections.
  • Progress indicators communicate task completion or progress.

Form and Selection Components

Selection controls allow users to choose options, filter information, or provide input within interactive experiences.

  • Forms collect user information.
  • Form groups organize related inputs.
  • Dropdown (Disclosure) components present selectable options.
  • Listboxes allow browsing and selection from lists.
  • Comboboxes combine text input with selectable options.
  • Search fields help users quickly locate content.

System Feedback Components

Feedback components communicate important information to users and confirm the results of interactions.

  • Alerts display important messages or warnings.
  • Alert dialogs present critical information that requires acknowledgment.

Dialog Interfaces

Dialogs present focused content or actions within a temporary interface layer. They may appear as modal or non-modal interactions depending on the context.

Advanced Interaction Patterns

Some interactive learning experiences require more complex interaction patterns.

  • Drag and Drop components allow users to move items within an interface.
  • Data charts and data visualizations communicate complex information visually.
