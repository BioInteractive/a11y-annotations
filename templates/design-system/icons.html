Design System Icons

Icon Library

The BioInteractive design system uses icons from the Untitled UI icon library. This icon set provides a consistent and modern visual language that supports clarity across the interface.

Untitled UI icons are designed on a consistent grid and follow standardized stroke styles, which ensures visual consistency across different interface components.

Icon Structure and Grid

All icons are designed on a 24×24 pixel grid with a consistent stroke width. This grid ensures that icons align cleanly with text and interface elements while maintaining visual balance.

  • Icons use a consistent 24×24 design grid.
  • Stroke-based icons maintain consistent visual weight.
  • Icons are optimized for use in both light and dark interface contexts.

Icon Categories

The icon library includes a wide range of categories that support common interface needs. These categories help designers quickly locate icons that match the meaning of a feature or interaction.

  • General interface icons
  • Navigation and arrow icons
  • User and communication icons
  • Files and development icons
  • Charts and data visualization icons
  • Education and science icons
  • Maps and travel icons
  • Weather and environment icons
  • Alerts and feedback icons
  • Media and device icons

Designers should select icons that clearly represent the intended action or concept.

Accessibility Considerations for Icons

Icons should enhance understanding and should not introduce confusion. Designers should ensure that icons are meaningful and easy to interpret within their context.

  • Meaningful icons should clearly represent the action or concept they support.
  • Icons must maintain sufficient contrast against their background.
  • Icons should remain visually clear at smaller sizes.

Decorative vs Informative Icons

Icons may be used either as decorative elements or as informative indicators. Designers should consider whether an icon conveys essential meaning.

  • Decorative icons are purely visual and do not convey essential information.
  • Informative icons communicate meaning and must be supported by accessible labels or text.

When icons communicate meaning, they should be paired with text labels or alternative descriptions so that users relying on assistive technologies can understand their purpose.

Consistent Icon Usage

To maintain a consistent interface, designers should use icons from the approved icon library rather than introducing custom icons. This ensures visual alignment across different components and projects.

Icons should be used sparingly and only when they enhance clarity or support a user action.
