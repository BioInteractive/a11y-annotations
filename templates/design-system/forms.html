Design System Forms

Forms Overview

Forms allow users to submit information, provide feedback, and interact with BioInteractive tools. The design system defines consistent patterns for form layout, labeling, validation, and user feedback to ensure forms remain clear, predictable, and accessible.

Following these patterns helps users understand what information is required, how to complete each step, and how to correct errors when they occur.

Form Structure

Forms should present fields in a logical order that matches the user’s mental model and reading flow. Related fields should be grouped together so users can easily understand how information is organized.

  • Use clear visual grouping to organize related form controls.
  • Arrange inputs in a logical top-to-bottom order.
  • Ensure form fields have consistent spacing and alignment.

Clear structure helps users scan the form quickly and complete tasks efficiently.

Form Groups

Form groups connect related inputs such as radio buttons, checkboxes, or related fields. These groups help users understand the relationship between multiple options.

  • Use labels and group headings to describe the purpose of each group.
  • Place group labels close to the related inputs.
  • Ensure grouped inputs remain easy to scan and understand.

Grouping controls reduces confusion and helps users make informed selections.

Labels and Instructions

Every form field must include a clear and visible label that describes the expected input. Labels help users understand what information is required and provide essential context for assistive technologies.

  • Place labels directly next to their associated input fields.
  • Provide additional instructions when formatting requirements exist.
  • Avoid relying solely on placeholder text for field labels.

Clear labels improve usability and reduce errors during form completion.

Status Messages

Status messages communicate important information about a form submission or system response. These messages should appear close to the relevant content and clearly indicate the outcome of an action.

  • Use consistent styling for success, warning, and error messages.
  • Place messages near the relevant form area.
  • Ensure messages remain visible and easy to understand.

Well-designed status messages help users understand what happened and what to do next.

Error Identification

When a form contains errors, the system should clearly identify which fields require correction. Error messages should appear near the associated input and provide clear instructions for resolving the issue.

  • Highlight fields that contain errors.
  • Provide concise error messages describing the issue.
  • Ensure error indicators remain easy to identify.

Clear error identification helps users correct mistakes quickly and complete the form successfully.

Error Suggestions

Error suggestions provide users with helpful guidance when correcting invalid inputs. Instead of simply indicating that something is wrong, the interface should explain how to resolve the issue.

  • Explain why the input is invalid.
  • Provide examples of valid input formats.
  • Offer guidance that helps users complete the form successfully.

Error Prevention

Where possible, forms should be designed to prevent errors before submission. Preventing mistakes reduces frustration and helps users complete tasks more efficiently.

  • Provide clear instructions before users enter information.
  • Use input formats that guide correct entry.
  • Offer confirmation steps for important actions.

Designing with error prevention in mind creates more reliable and user-friendly forms.

Identify Input Purpose

Forms that collect user information should clearly identify the purpose of each input field. Fields such as name, email, phone number, and address should be labeled consistently to support autofill and assistive technologies.

Grouping related fields and maintaining clear labels ensures that users understand what information is required and how it will be used.
