Accessibility Annotation Repository

This repository empowers Designers, Auditors and Developers to collaborate within Figma and streamline the process of creating born accessible webpages and components. Designers can reference this repository for accessibly guidance during their design process. Auditors may use this repository to help identify potential accessibility issues in the design phase and prepare annotations based on accessibility requirements, specifications, and best practices for a seamless developer hand-off. Developers may use the repository to gain insights into the auditor annotations and understand requirements for developing accessible web pages and components.

The repository offers comprehensive lists of applicable WCAG requirements and examples. The repository also offers thorough checklists for Designers, Auditors and Developers to use during thier individual processes. Resources are listed throughout the repository as well as in the resources section to assist in further exploration of concepts.

This repository references WCAG 2.1 AA success criteria requirements.

Navigating the Accessibility Annotation Repository

Overview

The Accessibility Annotation Repository is a dedicated space designed to enhance web accessibility through structured and focused auditing. The repository is organized into specific sections that align with key aspects of the Web Content Accessibility Guidelines (WCAG).

Repository Structure

The repository is divided into the following sections:

  • General: Includes key accessibility topics such as:
    • Structure
    • Color
    • Non-text Content
    • Interactivity
    • Screen Magnification
  • Components: Contains a list of complex interactive components requiring additional accessibility considerations.
  • Resources: Provides tools and guides for further support and exploration of accessibility concepts.

Components Section

Our Components section features an alphabetical list of components familiar to the HHMI development process, often requiring extra accessibility considerations. Each component page includes:

  • WCAG Success Criteria Requirements
  • Role, Property, State, and Tab Index Attribute Tables
  • Accessibility Considerations for Designers
  • How to Test and Annotate Section for Auditors
  • Accessibility Considerations for Developers

This structure makes the repository an indispensable resource for designers and developers, bridging the gap between accessible design intention and practical implementation.
